# CyberYARA

YARA rules for file and memory scanning of malicious artifacts. Generated and maintained by AgentForge from live threat intelligence. Companion to [CyberSigma](https://github.com/Vador05/CyberSigma) (log-based detection) and [CyberScripts](https://github.com/Vador05/CyberScripts) (Python analysis tools).

## When to use YARA vs Sigma

- **YARA** — scan files, binaries, memory dumps, package contents. *"Is this artifact malicious?"*
- **Sigma** — detect behaviors in SIEM logs. *"Did something suspicious happen?"*

## Usage

```bash
# Scan a single file
yara rules/pypi_trojan_credential_harvester.yar suspicious_package.py

# Scan recursively
yara -r rules/python_vuln_patterns.yar /path/to/codebase/

# Scan an extracted PyPI wheel
pip download requests --no-deps -d /tmp/pkg
cd /tmp/pkg && unzip requests-*.whl -d requests_extracted
yara -r rules/pypi_trojan_credential_harvester.yar requests_extracted/
```

## Rules

| Rule | Target | Threat | Severity |
|------|--------|--------|----------|
| [pypi_trojan_credential_harvester](rules/pypi_trojan_credential_harvester.yar) | Python source files / PyPI wheels | Supply chain trojan — env var exfil, C2 webhooks, encoded payloads | Critical |
| [python_vuln_patterns](rules/python_vuln_patterns.yar) | Python source files | SQLi, command injection, hardcoded secrets, path traversal, dangerous eval | High |
